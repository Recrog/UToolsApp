.gitignore Dosyası Oluşturucu

.gitignore dosyası, Git'in hangi dosya ve klasörleri takip etmemesi gerektiğini belirtir. Bu, gereksiz dosyaların (log dosyaları, derlenmiş kodlar, ortam değişkenleri vb.) depoya eklenmesini önler.

.gitignore Nasıl Oluşturulur?

İki ana yöntem vardır:

  1. Manuel Oluşturma: Proje kök dizininde ".gitignore" adlı bir dosya oluşturup içine kuralları yazabilirsiniz.
  2. Otomatik Oluşturma: Aşağıdaki araçla projenizde kullandığınız teknolojileri seçerek otomatik olarak oluşturabilirsiniz.
İpucu: GitHub, birçok programlama dili ve framework için hazır .gitignore şablonları sunar. Aşağıdaki araç bu şablonları kullanır.

Otomatik .gitignore Oluşturucu

Projenizde kullandığınız teknolojileri seçin:

Örnek .gitignore İçeriği

# Logs logs *.log npm-debug.log* yarn-debug.log* yarn-error.log* # Runtime data pids *.pid *.seed *.pid.lock # Dependency directories node_modules/ jspm_packages/ # Optional npm cache directory .npm # Optional REPL history .node_repl_history # Output of 'npm pack' *.tgz # Yarn Integrity file .yarn-integrity # dotenv environment variables file .env .env.test # parcel-bundler cache (https://parceljs.org/) .cache .parcel-cache # next.js build output .next # nuxt.js build output .nuxt # vuepress build output .vuepress/dist # Serverless directories .serverless # FuseBox cache .fusebox/ # DynamoDB Local files .dynamodb/
